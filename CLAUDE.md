## What This Is

A local, AI-powered web application (localhost only, single-user, V1) that turns video scripts into labeled, style-consistent visual asset packages. Input: `.srt` caption file or raw script text. Output: named PNG images + a distribution reference document.

## Architecture

**Decoupled client-server, localhost only.**

- `backend/` — Python 3.11+ / FastAPI, async throughout
- `frontend/` — Vanilla HTML/CSS/JS, no framework, no build step
- `outputs/` — all generated assets land here (gitignored)
- `backend/sessions/` — session state as JSON files

**Backend modules and their responsibilities:**

| File | Responsibility |
|---|---|
| `main.py` | FastAPI app entry point, all route definitions |
| `srt_parser.py` | SRT timecode/duration/segment parsing |
| `script_parser.py` | Raw script file + paste ingestion |
| `llm_service.py` | All Gemini API calls: mapping, prompt gen, scoring |
| `image_service.py` | Image generation abstraction layer — Kie.ai in V1, swappable |
| `scorer.py` | Quality scoring loop + auto-regeneration logic |
| `exporter.py` | File naming, output packaging, reference doc generation |
| `session_store.py` | In-memory session state |
| `style_library.json` | Persisted built-in + user-saved style presets |
| `schemas.py` | Pydantic models: `Scene`, `StyleConfig`, `Session` |

**Frontend** (`frontend/app.js`) handles: mode selection, file upload, chat UI for mapping review, style panel, generation progress, export trigger.

## Pipeline Flow

```
Input Upload → Parsing → LLM Mapping Generation
→ Chat Review (user edits) → Approval (locked)
→ Style Selection
→ Generation Engine: Prompt → Image → Score → Retry
→ Export (images + reference.md)
```

**Two input modes diverge only at analysis; everything downstream is identical:**

- **Mode A (SRT):** Duration-based scene type heuristics (< 3s → TITLE/DESCRIPTOR, 3–6s → HYBRID/INFOGRAPHIC, > 6s → INFOGRAPHIC)
- **Mode B (Raw Script):** Semantic importance-based; requires user to supply approximate video duration as a soft calibration target

---

## Key System Rules

**Scene types:** `INFOGRAPHIC`, `TITLE`, `DESCRIPTOR`, `HYBRID` — fixed in V1. INFOGRAPHIC is always standalone (no text overlay). Text-based types (TITLE/DESCRIPTOR/HYBRID) have all text generated by the image API.

**Aspect ratio:** LLM assigns — 1:1 for comparisons/text scenes, 16:9 for full explanations.

**Background:** Solid color only (preset or hex). No gradients. Enforced at prompt level AND validated post-generation.

**Scoring (hidden from UI):** Threshold is 7.5. Hard fails: wrong text, blank image, wrong subject, wrong background. Max 3 retry attempts per image. On 2nd failure: pause and show user both outputs. On 3rd failure: mark scene `failed`, continue pipeline.

**Editing:** No edits mid-generation. After completion, `/regenerate-scenes` with specific scene IDs is allowed. Stop/resume (from start or from current scene) is allowed during generation.

**Session state:** Sessions persisted as JSON under `/sessions/`. Open question: localStorage vs SQLite — decide before implementing Phase 3.

## API Endpoints

| Method | Endpoint | Purpose |
|---|---|---|
| POST | `/upload` | SRT file or script text, mode, optional duration |
| POST | `/generate-mapping` | Trigger LLM scene distribution |
| POST | `/chat` | Conversational mapping edits |
| POST | `/approve-mapping` | Lock the plan |
| POST | `/generate` | Start image generation (`batch_size`: 1/5/10) |
| POST | `/stop` | Stop generation |
| POST | `/resume` | Resume (`mode`: `from_start` or `from_current`) |
| POST | `/regenerate-scenes` | Re-run specific scenes post-completion |
| GET | `/status` | Poll generation progress |
| POST | `/export` | Package assets + generate reference doc |

## Configuration

```
.env               # GEMINI_API_KEY, KIE_API_KEY — never committed
style_library.json # style presets, loaded on startup
outputs/           # gitignored
sessions/          # gitignored
```

**LLM:** Google Gemini via `google-generativeai` SDK. Recommended model: `gemini-2.5-flash` (confirm at build time).

**Image generation:** Kie.ai REST API in V1 — 80 free credits for testing. All calls go through `image_service.py` so the provider is swappable by changing one file.

**Concurrency:** Batch size up to 10, parallel workers: 3.

## Phased Build Order

1. Input & Parsing (SRT + raw script, mode selection UI)
2. LLM Mapping via Gemini (display only, no chat)
3. Chat Review Interface + approval locking
4. Style Configuration panel + style library persistence
5. Generation Pipeline (image gen, scoring, auto-regeneration)
6. Export (file naming, packaging, reference doc)

Each phase is independently testable before the next begins.

## V2 Scope (Deferred — Do Not Design For)

Video animation (Higgsfield/Seedance), cloud hosting, multi-user, Premiere Pro integration. V1 image exports should use **1:1 aspect ratio** to remain forward-compatible with V2 animation without re-generation.
